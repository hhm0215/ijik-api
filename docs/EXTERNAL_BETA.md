# 외부 실사용 베타 운영 기준

현재 API는 외부 HTTPS로 접근 가능한 **개발·실사용 검증 환경**이다. 아직 사용자
계정과 권한이 없으므로, 이 문서의 보호 조치 전에는 불특정 다수에게 제품으로
알리거나 실제 개인정보를 입력받지 않는다.

## 단계

1. 본인 실사용: 프론트 화면을 API와 연결하고 실제 데이터로 매일 사용한다.
2. 제한 베타: 로그인·권한, 백업, 오류 추적을 구현한 후 초대받은 테스터만 연다.
3. 공개: 지표와 피드백을 바탕으로 필요한 제품 기능만 확장한다.

## VPS 우선 조치

호스트 Ollama의 `11434/tcp`가 외부에 노출된 상태다. OpenClaw가 호스트 Ollama를
호출하는 경로를 먼저 확인한 다음에 차단해야 한다. 호스트에서 다음을 실행한다.

```bash
ss -tlnp | grep 11434
docker ps --format 'table {{.Names}}\t{{.Ports}}'
docker inspect <openclaw-container-name> --format '{{json .HostConfig.ExtraHosts}}'
```

컨테이너가 호스트 Ollama를 호출하지 않는 것이 확인되면, Ollama를 `127.0.0.1`에만
바인딩하거나 VPS/호스팅 방화벽에서 외부 `11434/tcp`를 차단한다. 차단 뒤에는
외부 네트워크와 OpenClaw 양쪽에서 동작을 확인한다.

## 프론트 연동 전 결정

- 프론트 저장소와 배포 도메인
- 첫 베타 사용자가 본인만인지, 초대 테스터가 포함되는지
- 인증 방식: 자체 계정/세션 또는 외부 인증 공급자

브라우저 코드에 고정 API 키를 넣어 쓰기 API를 막는 방식은 비밀값이 노출되므로
제한 베타의 인증 방식으로 사용하지 않는다.
